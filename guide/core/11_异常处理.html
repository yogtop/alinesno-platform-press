# 服务异常

服务异常统一由后台抛出，由前端获取，Dubbo 抛出异常的形式为String，此处由统一处理之后，再抛出。

使用RpcServiceRuntimeException异常对象

例如:

try{
    // 业务处理
} catch (Exception e) {
    log.debug("短信发送失败:{}", e);
    throw RpcServiceRuntimeException.error("短信发送失败:" + e);
}

# 前端异常

前端异常统一返回ResponseBean实体，结构为:

// 成功则code = 200

{
  "code": 400,
  "message": "对用户[333]进行登录验证..验证未通过,错误的凭证",
  "data": null
}
